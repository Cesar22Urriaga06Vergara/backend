Contexto del proyecto

Una respuesta tecnológica para hoteles que necesitan ordenar su operación

El software nace como una solución académica orientada a resolver necesidades frecuentes en hoteles: consultar reservas, registrar entradas y salidas, administrar habitaciones, controlar servicios, revisar facturación y entregar información clara a cada usuario según su rol.

ADUS Hospitality OS fue desarrollado por aprendices del programa Técnico en Desarrollo de Software, en el marco del proceso formativo comprendido entre julio de 2025 y junio de 2026. El equipo de trabajo enfocó el proyecto en una necesidad concreta: contar con un sistema que reúna en un solo entorno digital las actividades principales de un establecimiento hotelero.

La plataforma permite que el administrador tenga control sobre los componentes esenciales del negocio, que el recepcionista ejecute con mayor claridad las tareas del día y que el cliente consulte información relacionada con reservas, servicios, pedidos y cuenta personal. Esta separación por perfiles fortalece el orden, la trazabilidad y la seguridad funcional del sistema.

Más que una interfaz de consulta, ADUS Hospitality OS propone una forma organizada de conectar personas, procesos e información dentro de la operación hotelera.

El proyecto integra elementos de análisis de requerimientos, diseño de experiencia de usuario, desarrollo de módulos, construcción de paneles, validación de flujos y presentación de resultados. Por ello, funciona como evidencia del aprendizaje técnico aplicado a un escenario cercano al sector productivo.

Funcionamiento por perfiles

Paneles diseñados para las responsabilidades de cada usuario

El sistema organiza la experiencia mediante roles. Cada perfil encuentra accesos y acciones relacionadas con sus tareas, evitando que todos los usuarios vean la misma información o dependan de menús extensos.

Administrador

Cuenta con el panel de mayor alcance. Desde allí puede gestionar usuarios, reservas, habitaciones, tipos de habitación, amenidades, servicios, facturas, reportes, supervisión de áreas y reinicio de estadísticas operativas cuando el flujo lo requiera.

Recepcionista

Dispone de herramientas para el trabajo diario de recepción: caja y folios, check-in, check-out, registro de incidencias, revisión de pendientes, control de salidas y estado operativo de la jornada.

Cliente o huésped

Accede a un entorno simple para consultar su información personal, revisar facturas, solicitar servicios, ver pedidos, crear o gestionar reservas y conocer el estado de su cuenta dentro del hotel.

Funcionalidades del sistema

Módulos que conectan administración, operación y atención al huésped

ADUS Hospitality OS reúne las funciones necesarias para acompañar el ciclo completo de una estadía: desde la configuración del hotel hasta la solicitud de servicios y el cierre de información operativa.

Reservas

Consulta, registro y seguimiento de reservas para organizar disponibilidad y ocupación.

Habitaciones

Administración de habitaciones, tipos, características y elementos asociados a la estadía.

Usuarios

Gestión de cuentas y perfiles para separar permisos de administrador, recepcionista y cliente.

Facturación

Consulta y organización de facturas para apoyar el cierre de consumos y cobros.

Servicios

Catálogo para solicitudes como alimentos, spa u otros servicios complementarios del hotel.

Amenidades

Registro y control de elementos que enriquecen la experiencia de alojamiento.

Reportes

Información resumida para analizar actividad, operación y comportamiento del sistema.

Control operativo

Seguimiento de incidencias, caja, folios, estados y tareas pendientes de recepción.

Base técnica del desarrollo

Una arquitectura web organizada en frontend y backend

De acuerdo con la documentación técnica del proyecto, ADUS Hospitality OS está construido como una aplicación web SPA conectada a una API REST. Esta separación permite que la interfaz por roles consuma servicios del backend de forma ordenada y que cada módulo mantenga responsabilidades claras.

Frontend SPA

La interfaz está desarrollada con Nuxt 3, Vue 3, TypeScript, Vuetify 3, Tailwind CSS, Pinia y Vite. Su configuración funciona como SPA, con rutas por páginas, layouts por rol, estado global y llamadas HTTP centralizadas mediante composables.

Nuxt 3 Vue 3 TypeScript Vuetify Tailwind CSS Pinia

Backend API REST

El backend está construido con NestJS 11, TypeScript, TypeORM y MySQL. Expone servicios para autenticación, usuarios, hoteles, reservas, check-in/check-out, folios, pagos, caja, facturación, servicios, reportes, incidencias y administración de plataforma.

NestJS TypeORM MySQL JWT Google OAuth Swagger

Flujo de trabajo

De la reserva al cierre de la estadía

El sistema acompaña diferentes momentos de la operación, permitiendo que la información avance entre áreas sin depender de registros aislados.

1

Configuración y administración

El administrador prepara usuarios, habitaciones, tipos de habitación, amenidades y servicios disponibles.

2

Reserva y llegada

La recepción revisa pendientes, valida información y registra el check-in del huésped.

3

Consumo de servicios

El cliente puede consultar accesos, solicitar servicios, revisar pedidos y hacer seguimiento a su cuenta.

4

Salida y análisis

El recepcionista gestiona check-out, caja y folios, mientras el administrador puede consultar reportes.
